第一次总结（9.5 ~ 9.10）

​ 这是新学期的第一周，也是我开始在宁波风链干活的第一周，初到陌生的环境感觉还是会有些不适应，但总归下来，这里的氛围是挺好的，也学到了不少的东西，因为之前提前学习了 go 的语法（虽然是速成），所以这周就直接开始看 gin 框架的用法了，主要就是看煎鱼佬的博客一起做项目，之后也会新建一篇文章总结概括一下，不过可能学的内容有点多，掌握可能不是那么熟练，许多官方文档还是要多看多用，很多用到的内容都没有细学，这是不足的地方，接下来简单概括下这一周的学习内容（主要以煎鱼的博客为主）:

  • Linux 环境下安装 Go ，配置编译器以及 Go module 包管理
  • Gin 安装及使用
  • 使用 Gin 搭建 Blog APIs
  • go-ini 读写配置文件
  • com 包的字符串转换
  • validation 包的表单验证
  • jwt-go 进行身份校验
  • 自定义 log
  • endless 来实现服务重新启动的零停机
  • 使用 swagger 生成接口文档
  • 将应用部署到 docker （没看懂）
  • 定制 GORM Callbacks

​ 虽然似乎内容不少，但是忘记也是很快的，令我担心的一点是学习的都不算深入，无法完全不参照的完成自己的独立内容，提到的很多文档都没有细看，另外也暴露了在某一些方面的不足比如说数据库表与表的联系这一块，学习并非只是看到一个结果，要理解其本质才能说是化为己用。另外也有不少内容没有完成，值得反思，比较严重的为以下几点

  • GORM 文档的详细阅读及笔记
  • docker 功能的了解（部署那一章基本没看懂）
  • git 工具的较为复杂的使用技巧（分支管理之类 / Sourcetree 之类可视化软件的使用）
  • 区块链基础课程还没看
  • 虽说是照着别人博客看，但自己也得记录一些内容
  • 空闲的时间太摆了（打算看的书没开看之类，嗯打游戏）

​ 煎鱼佬的博客毕竟就在那里，我再抄一遍也没多少大意义，简单摘录几句，这两天读下文档，复习下基础吧

第二次总结（9.12 ~ 9.18）

​ 这是开学第二周，也是上班的第二周，虽说中秋放假休息了一天，这周做的东西明显没有上周来的多，分析其原因主要还是环境的问题，配了两次环境一次在Centos 7上部署，一次在 Centos 8上部署结果遇到的问题却有点不一样，这个拖慢了节奏，不过这个问题已经解决了，在得到可以将公司电脑长开机后，之后不会存在需要部署同样的内容两趟的情况，这对之后的工作来说是一个巨大的提升。接下来简单说说这周都做了些什么，实际上在周记上已经写出来了，但是这边还是再列举一遍，本周完成的工作主要有以下几个：

  • Geth的学习和使用
  • 使用 Geth 在Centos 8操作系统下搭建私链
  • 使用 Geth 提供的Console 在搭建的私链上操作
  • 使用 Go-ethereum 对私链和 Goerli 测试链进行操作
  • 尝试调用 alchemy 提供的部分 API 对 Goeril 测试链进行操作

​ 虽然看起来没有上周的内容来的多，但是实际上遇到的问题确实有一些，当然我认为最重要的原因还是台风在家办公的一天，没有在这边的机子上配好对应的环境导致的，当然这个问题也已经解决了，之后还是惯例咱列举下下周打算做的事情：

  • 学会 Go-ethereum 上与合约操作相关的内容
  • alchemy 提供的大部分API 调用的学习
  • 整理学习阶段的各个知识点
  • 如果完成速度快可能开始源码阅读
  • 私下会可能抽空学习一点Solidity编程

​ 在我看来明面上来说区块链基础的学习部分已经完成了一个小阶段，但我清楚我在之前的学习过程中，肯定是存在某些不完美的情况的，然而这种问题究竟出现在何处，除了需要自己仔细思考反思以外，还是需要多一些练习的，掌握一门技术光光可以看懂是不够的，何况我可能也看不懂。所以下周我的第一个打算就是对现在已经学的知识做一个系统性的总结。当然对以太坊的基础理解上，部分理解现在已经不适用了，另外还有一部分原因是因为可能有些知识已经开始遗忘，学习的进度需要加紧，所以打算提出先看源码的方案，学习在实际项目中 gin 框架、go-ethereum 以及alchemy 提供的API是怎样进行有机的交互，另外因为 ERC20 的代币发行会用到 Solidity 相关的知识，所以我打算学习一部分智能合约相关的内容，选用课程是（BV1ES4y1J7FB），这些目标可能会有点多，我也不清楚一周时间是否足够，因此可能会存在无法完成的情况，还望莫怪。

​ 讲一些我心里的想法，我现在处于一个知道之后要学啥的状态，但是可能也正是因为我知道我要学的内容有很多，搞得自己有点焦虑，虽然我清楚这显然是没有必要的，但总归对心态还是有点影响，我也觉得自己太急了，可能稍微缓一些会好一点，狼吞虎咽没什么大的好处，只会紧逼自己。我对这个行业还是挺喜欢的，起码觉得挺有意思的，也许正是这种想要将自己推进行业里的心态，所以才那么急吧，可是道理大家都明白，有很多要学的内容的时候可能就摆烂啥都不学了，这就哈人了。没必要把学习当做一个被动的事情。挺无语的，我也不知道我在急啥，但就是很急。。。

​ 一些题外话，最近中午老是困，但我其实很不愿意午休，看样子下周开始下午得买杯可乐充电了。得搬个位置，拖好久了，周一要早点到。其实我挺喜欢与人交谈吹水，但我不喜欢主动和不熟的人交谈，被动社恐是这样的。想学英语，地铁上背背单词吧。最后就是得锻炼下了，从下周骑车回家开始吧，体力太杂鱼了，科学养猪，科学养猪。。。

第三次总结（9.19 ~ 9.25）

​ 三周了，现在是2022年9月25日22:57:37，想要上床躺着了，所以长话短说，先copy一下周结和下周安排

本周完成工作：

  • 完成了合约操作相关的Goethereum 上的全部样例，除去最后两块内容
  • alchemy上的绝大多数API成功调用
  • 阅读O3Hub 的源码,了解了部分代码的工作机制对gin业务的实际使用有了一个初步的认识
  • 私下学习了一点Casbin使用，毕竟提到了权限

下周工作计划：

  • 听从安排，进行任务
  • 没有安排，咱就先随意玩玩，可能会把用户模块按自己理解操作一些。也可能会把 casbin 相关的 api 给他写出来，不过具体怎么搞还是要问问，自己瞎搞可能没有意义。

​ 最后还是讲一些内心想法，哥们儿觉得无问题啊，虽然总感觉一周写的东西比一周少了（指文档），不过，这周周日难得学了点东西，没有躺平摆烂，值得庆祝，值得庆祝。我还是想法很多，一些我想做的，不过怕说出来做不动，还是先做在说出来，虽然哥们儿话不多但是计划不少。咱这一周五天虽然痛苦但每天都骑10公里回学校，握力器每天玩个爽，雅思词汇嗯背20个，上周的计划咱都兑现了，不错的不错的。就是胖了一斤是真的郁闷。不过其他一些能做的也是让我有些惭愧，《100 Go Mistakes …啥的名字一大堆》，这书不错，就是我读的似乎确实有点好慢，一步一步来吧，这书700页，咱读完了概述和第一个问题，还有99个，接下来也许会关于这个写篇读书笔记啥的，也可能不会写，会写在notability上也说不定，还蛮有成就感的，我认为人生也该如此，保持好奇，然后获得成就。

第四次总结（9.26~10.07）

​ 国庆过完了，我感觉我就是一条臭咸鱼，啥事也干不成。虽说是放假但似乎是迷迷糊糊吃吃喝喝就过去了，长久的空闲带来的不小的空虚感，纯纯的Five。也是需要收下心来进行后续的工作和生活，可能我需要对自己的行为、态度、思想做一次深刻的自我检视，感觉问题会不小，这些内容我不打算公开所以不会上传到公开的博客中，我需要自己仔细掂量自己的未来的思路。

​ 我不清楚这算不算是工作上的失误，不过确实上周的任务在我看来没有顺利的完成，应该也没有完成老大的期待，本来打算趁着国庆花上一些时间来完成这件事的，结果只抽出了差不多一天来做这件事，道心不坚啊，小友。

​ 更多的文字就不再这里写了，我确实需要更加深入的思考。但这不该是一个找借口的过程，我承认这是我的过失，可能还是我的内心太过于浮躁了，此外还有一些其他的内容需要思考，希望即将到来的7日连上能够顺利吧。

本周完成工作：

​ 见随手来一句，文字较多，放在末尾了

下周计划工作：

​ 我认为我得向老大说明下我的状况，没有必要刻意隐瞒，老大可能想让我开始接触实际的业务代码了，但是确实我还有之前欠下的一些内容没有做完，说到底还是我自己这些天太浮躁了。希望我还有机会可以弥补这个过失。
