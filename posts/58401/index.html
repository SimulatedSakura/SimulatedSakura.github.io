Golang 课程序言

Golang的优势

  1. 极简单的部署方式

    1. 可直接编译成机器码
    2. 不依赖其他库
    3. 直接运行可部署

  2. 静态类型的语言

    1. 编译的时候检查出来隐藏的大多数问题

  3. 语言层面的并发

    1. 天生的基因支持

    2. 充分的利用多核

  4. 强大的标准库

    1. runtime 系统调度机制
    2. 高校的 GC 垃圾回收
    3. 丰富的标准库

  5. 简单易学

    1. 25关键字
    2. C 语言简洁基因 ， 内嵌C语法支持
    3. 面向对象特征（继承、多态、封装）
    4. 跨平台

Golang的强项

  1. 云计算基础设施领域（docker、k8s）
  2. 基础后端软件（docker、k8s）
  3. 微服务（go-kit）
  4. 互联网基础设施（以太坊、hyperledger）

Golang的明星作品（成就）

​ Docker Kubernetes

Golang 的不足

  1. 包管理,大部分过都在 GitHub 上
  2. 无泛化类型（1.14）
  3. 所有的 Exception 都用 Error来处理(比较有争议)
  4. 对 C 的降级处理, 并非无缝, 没有 C 降级到 asm (汇编) 那么完美(序列化问题)
